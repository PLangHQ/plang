# Phase 0.3 — `/system/error/` Layer

## Goal
Create PLang goals and HTML templates for structured error display. Replace the current minimal error goals (`write out system error "%!error%"`) with proper error formatting that uses the `ErrorCategory` from Phase 0.2.

## Depends On
- Phase 0.2 (ErrorCategory enum) must be complete — ConsoleError.goal uses `%!error.Category%`

## Current State
Error display is handled by three minimal event goals:
- `system/events/Runtime/OnAppError.goal` → `write out system error "%!error%"`
- `system/events/Runtime/OnStepError.goal` → `write out system error "%!error%"`
- `system/events/Runtime/OnGoalError.goal` → `write out system error %!error%`

These just dump the raw `Error.Format()` output. No distinction between Application and Runtime errors.

## Changes

### 1. Create `system/error/ConsoleError.goal`
```plang
ConsoleError
/ Formats %!error% for console display
/ Application errors get concise output; Runtime errors get full detail
- if %!error.Category% = "Application" then
    - write out "Error: %!error.Message%"
- if %!error.Category% = "Runtime" then
    - write out "%!error.Format%"
```

### 2. Create `system/error/error.html`
Generic HTML error template (Scriban syntax) for future web server use:
```html
<!DOCTYPE html>
<html><head><title>Error</title>
<style>
body { font-family: system-ui; max-width: 800px; margin: 40px auto; padding: 0 20px; }
.error { background: #fef2f2; border: 1px solid #fca5a5; border-radius: 8px; padding: 20px; }
.error h1 { color: #dc2626; margin-top: 0; }
.detail { color: #6b7280; font-size: 0.9em; margin-top: 12px; }
pre { background: #f3f4f6; padding: 12px; border-radius: 4px; overflow-x: auto; }
</style></head>
<body>
<div class="error">
  <h1>{{ error.Key }} ({{ error.StatusCode }})</h1>
  <p>{{ error.Message }}</p>
  {{ if error.Step }}
  <div class="detail">
    <p><strong>Step:</strong> {{ error.Step.Text }}</p>
    <p><strong>File:</strong> {{ error.Step.Goal.Path }}:{{ error.Step.LineNumber }}</p>
  </div>
  {{ end }}
  {{ if error.FixSuggestion }}
  <div class="detail"><p><strong>Suggestion:</strong> {{ error.FixSuggestion }}</p></div>
  {{ end }}
</div>
</body></html>
```

### 3. Create `system/error/500.html`
System error HTML template with full detail:
```html
<!DOCTYPE html>
<html><head><title>Internal Error</title>
<style>
body { font-family: system-ui; max-width: 800px; margin: 40px auto; padding: 0 20px; }
.error { background: #fef2f2; border: 1px solid #fca5a5; border-radius: 8px; padding: 20px; }
.error h1 { color: #dc2626; margin-top: 0; }
.detail { color: #6b7280; font-size: 0.9em; margin-top: 12px; }
.stack { font-size: 0.85em; color: #374151; }
pre { background: #f3f4f6; padding: 12px; border-radius: 4px; overflow-x: auto; }
</style></head>
<body>
<div class="error">
  <h1>Internal Error</h1>
  <p>{{ error.Message }}</p>
  {{ if error.CallFrames.size > 0 }}
  <div class="stack">
    <h3>Call Stack</h3>
    <ul>
    {{ for frame in error.CallFrames }}
      <li>{{ frame.GoalName }} - {{ frame.GoalPath }}{{ if frame.Step }}:{{ frame.Step.LineNumber }}{{ end }}</li>
    {{ end }}
    </ul>
  </div>
  {{ end }}
  {{ if error.Exception }}
  <div class="detail">
    <h3>Exception</h3>
    <pre>{{ error.Exception.Message }}
{{ error.Exception.StackTrace }}</pre>
  </div>
  {{ end }}
</div>
</body></html>
```

### 4. Update event error goals to delegate to ConsoleError

**`system/events/Runtime/OnAppError.goal`** — Change to:
```plang
OnAppError
- call /system/error/ConsoleError
```

**`system/events/Runtime/OnStepError.goal`** — Change to:
```plang
OnStepError
- call /system/error/ConsoleError
```

**`system/events/Runtime/OnGoalError.goal`** — Change to:
```plang
OnGoalError
- call /system/error/ConsoleError
```

### Note on HTML templates
The HTML files are static templates for Phase 6 (web server). They won't be rendered now — they exist to establish the pattern and directory structure. When the web server module is built, the error event goals will detect the Actor type and either call ConsoleError or render the appropriate HTML template.
