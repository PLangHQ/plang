# Phase 1.3 -- Retry Logic

**Status:** NOT STARTED
**Complexity:** Small
**Depends on:** Phase 0

## Goal
Add retry capability to Step execution on error.

## Design
- **Not a module** - it's Step infrastructure (like Cache)
- **Add:** `Step.RetrySettings` (maxRetries, delayMs, backoffMultiplier)
- **Integration:** StepMethods.RunAsync checks retry on Data.Error()

## Files
- Modify: `PLang/Runtime2/Core/Step.cs` - add RetrySettings
- Modify: `PLang/Runtime2/Core/StepMethods.cs` - retry loop on error

## Tests
- C# unit tests: retry count, backoff timing, eventual success/failure

## Verification
- Step retries up to maxRetries on error
- Backoff multiplier increases delay between retries
- Successful retry returns Data.Ok()
