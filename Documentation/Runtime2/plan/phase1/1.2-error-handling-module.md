# Phase 1.2 -- Error Handling Module (`error/throw`)

**Status:** NOT STARTED
**Complexity:** Medium
**Depends on:** Phase 0

## Goal
Allow PLang developers to throw intentional, user-facing errors.

## Design
- **Handler:** `error/throw.cs` - PLang developer throws an error
- **Parameters:** message (string), statusCode (int?), type (string?)
- **This is a PLang developer error** - intentional, user-facing (e.g., validation failure)
- Uses `Data.FromError()`
- **Step-level:** Wire up Step.OnErrorGoal to catch and handle errors
- **Error display:** Delegates to `/system/error/` PLang goals for rendering

## Files
- New: `PLang/Runtime2/modules/error/throw.cs`
- Modify: Step error handling infrastructure

## Tests
- C# unit tests: throw with various parameters, error propagation
- PLang integration: `Tests/Runtime2/ErrorHandling/`

## Verification
- Thrown errors propagate correctly through Step -> Goal -> Engine
- OnErrorGoal catches thrown errors
- Error display via `/system/error/` works
