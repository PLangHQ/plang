From idea to running software.
No setup ceremony.

Install PLang, write a .goal file in plain English, and run it. Your first app takes about 60 seconds.

1

Install PLang

Download the latest release for your platform from GitHub Releases:

PlatformDownload
Windowsplang-windows.zip
Linux (x64)plang-linux-x64.zip
Linux (ARM)plang-linux-arm64.zip
macOS (Intel)plang-osx-x64.zip
macOS (M-series)plang-osx-arm64.zip

Unzip and add the folder to your PATH. Verify:

$ plang --version
2

Create your first app

$ mkdir HelloWorld && cd HelloWorld

Write your program in Start.goal:

Start.goal
Start- write out 'Hello PLang world'

That's the whole program. One goal, one step.

3

Build and run

$ plang exec
Hello PLang world

This builds (sends your step to the LLM, compiles it into a .pr file) and runs (executes the compiled instruction).

On your first run, you'll be prompted to set up a PLang service account or OpenAI API key. A $5 credit goes a long way — building each step costs $0.002–$0.009.

4

Build something real

Create a Start.goal that actually does something:

Start.goal
Start- insert into tasks, description='Buy groceries', done=false, write to %id%- insert into tasks, description='Read PLang docs', done=false- select * from tasks, write to %tasks%- write out %tasks%
$ plang exec

What just happened:

  • PLang created a SQLite database automatically (no config, no Docker)
  • Inserted two rows into a tasks table (created on first insert)
  • Queried all tasks and displayed them

You didn't install a database. You didn't write a schema. You didn't configure a connection string.

5

Add more power

A task manager with an API and email notifications:

Start.goal
Start- start webserver
AddTask - POST- insert into tasks, description=%request.description%,    assignee=%request.assignee%, done=false, write to %id%- send email to %request.assignee%, subject "New Task",    body "You've been assigned: %request.description%"- write out {id: %id%, status: 'created'}
ListTasks - GET- select * from tasks where done=false, write to %tasks%- write out %tasks%
CompleteTask - POST- update tasks set done=true where id=%request.id%- write out {status: 'completed'}

You now have a running web server on http://localhost:8080 with three API endpoints, a database, and email notifications. In 15 lines of English.

Project structure

HelloWorld/
  Start.goal ← Your source code
  .build/
    Start/
      00. insert into tasks.pr
      01. insert into tasks.pr
      02. select from tasks.pr
      03. write out.pr
  .db/
    system.sqlite
    data.sqlite
  • .goal files — your source code. Version these.
  • .build/ folder — compiled JSON instructions. Inspect to verify what the LLM produced.
  • .db/ folder — SQLite databases. Created automatically.

Honest notes

  • Building costs money — LLM calls at build time cost $0.002–$0.009 per step. A $5 credit lasts a long time.
  • Runtime is free — no AI calls, no network, no ongoing cost.
  • PLang is young — best suited for internal tools, automation, and prototypes today.
  • You can inspect everything — .pr files are JSON. If the LLM compiled something wrong, fix the wording, rebuild, done.
