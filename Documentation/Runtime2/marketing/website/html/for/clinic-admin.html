Appointments and reminders,
handled.

The phone rings. Someone wants to reschedule. You open the calendar, find an open slot, update the booking, then remember to text the doctor. Meanwhile, three patients from yesterday didn't show — nobody sent them a reminder. You're managing appointments in one system, patient contacts in another, and sending reminders manually when you remember.

The PLang way

Start.goal
Start- start webserver- every day at 4pm, call !SendTomorrowReminders- every monday at 8am, call !WeeklyNoShowReport
BookAppointment - POST- select count(*) as conflicts from appointments    where doctor=%request.doctor% and date=%request.date%    and time=%request.time%, write to %check%- if %check.conflicts% > 0 then    - write out {error: 'Time slot taken'}, status code 409- insert into appointments, patient=%request.patient%,    patientEmail=%request.email%, doctor=%request.doctor%,    date=%request.date%, time=%request.time%,    status='confirmed', write to %id%- send email to %request.email%,    subject "Appointment Confirmed",    body "Your appointment with Dr. %request.doctor% is confirmed for %request.date% at %request.time%."- write out {appointmentId: %id%, status: 'confirmed'}
SendTomorrowReminders- select * from appointments where date=%Tomorrow%    and status='confirmed', write to %tomorrowAppts%- foreach %tomorrowAppts%, call !SendReminder item=%appt%
SendReminder- send email to %appt.patientEmail%,    subject "Reminder: Appointment Tomorrow",    body "Hi %appt.patient%, reminder that your appointment with Dr. %appt.doctor% is tomorrow at %appt.time%."
MarkNoShow - POST- update appointments set status='no-show'    where id=%request.appointmentId%- write out {status: 'marked no-show'}
WeeklyNoShowReport- select patient, count(*) as noShows from appointments    where status='no-show' and date >= %Now-7days%    group by patient order by noShows desc, write to %report%- send email to admin@clinic.com,    subject "Weekly No-Show Report",    body "No-shows this week:\n%report%"
DailySchedule - GET- select * from appointments where date=%Today%    order by time, write to %schedule%- write out %schedule%

That's your clinic scheduling system.

Appointment booking with conflict checking, automatic reminders, no-show tracking, weekly reports. Patients get confirmations. You get a dashboard.

What just happened

  • insert into appointmentsAppointment database created automatically. No software subscription needed.
  • select count where doctor and date and timeDouble-booking prevention built in. Conflicts caught before they happen.
  • every day at 4pmReminders sent automatically every afternoon. No more no-shows from forgetfulness.
  • send emailConfirmations and reminders sent automatically. Professional and consistent.
  • WeeklyNoShowReportTrack patterns. Know which patients need a phone call instead of an email.
  • DailySchedule - GETToday's schedule accessible from any browser. Share it with the whole office.

Need to send reminders at 6pm instead of 4pm? Change one line. Need to add SMS? Add one step.

The build / run split

PLang uses AI at build time to compile your English into executable instructions. At runtime, no AI — your scheduling system runs reliably with zero ongoing cost. Build once, your clinic operations run forever.
