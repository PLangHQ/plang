# Hello World

Let's write, build, and run your first PLang program.

## 1. Create a Project Folder

```bash
mkdir HelloWorld
cd HelloWorld
```

## 2. Write a Goal File

Create `Start.goal`:

```plang
Start
- write out 'Hello PLang world'
```

That's the entire program. `Start` is the goal name (the entry point). The step `write out` sends text to the output.

## 3. Build and Run

```bash
plang exec
```

On the first run, PLang builds your code and then executes it. You'll see:

```
Hello PLang world
```

## What Just Happened?

1. PLang read `Start.goal`
2. The builder sent each step to an LLM, which mapped `write out` to the `output/write` action
3. A compiled `.pr` file was written to `.build/Start/00. Start.pr`
4. The runtime executed the `.pr` file — no LLM needed

## A Slightly Bigger Example

Create `Start.goal`:

```plang
Start
- set %name% = 'PLang'
- set %version% = 1
- write out 'Welcome to %name% version %version%'
- if %version% equals 1 then call FirstVersion

FirstVersion
- write out 'You are running the first version!'
```

Run it:

```bash
plang exec
```

Output:

```
Welcome to PLang version 1
You are running the first version!
```

This shows variables (`%name%`), conditions, and goal calls — the building blocks of every PLang program.

## Project Structure After Build

```
HelloWorld/
├── Start.goal          ← your source code
└── .build/
    └── Start/
        └── 00. Start.pr   ← compiled instructions
```

The `.build/` folder contains compiled `.pr` files. You never edit these — they're generated by the builder.
